# Security Policy

## Reporting a vulnerability

Do not open public issues for security vulnerabilities.

Contact maintainers privately at: [contato@jorgeferreira.com.br](mailto:contato@jorgeferreira.com.br)

This process may change as Polaris grows and individual repositories define their own security policies.

---

## Scope

This policy applies to all public Polaris repositories. Many repositories are early drafts, experiments, or examples — but security concerns still matter regardless of maturity stage.

---

## Do not include sensitive data

Never include the following in issues, PRs, discussions, commits, or prompts:

- Secrets or credentials
- API keys or tokens
- Private logs
- Customer data
- Corporate data
- Real personally identifiable information (PII)

If you discover sensitive data already committed to a repository, report it privately rather than opening a public issue.

---

## Learning projects and synthetic data

Public learning projects must use synthetic data by default.

Examples involving incidents, logs, support tickets, payments, orders, users, or operational data must be clearly synthetic unless explicitly documented otherwise in the repository.

---

## Supported versions

Polaris is early-stage. Most repositories do not have stable releases. Security support is best-effort until individual repositories define their own policies.
