# Support

Polaris is an open community in an early stage. There is no dedicated support team. Maintainers and contributors help when they can.

---

## Where to ask

1. **Read the repository README first.** Most repositories document their purpose, status, and how to use them.
2. **GitHub Discussions** — if enabled in the repository, use it for questions and open-ended topics.
3. **GitHub Issues** — for bugs, unclear documentation, broken examples, reproducibility problems, or missing instructions.

---

## What to include when asking

Clear questions with context are easier to answer. Include:

- What you are trying to do
- What you expected to happen
- What actually happened
- Steps to reproduce the issue
- Relevant command output or error messages
- What you already tried

---

## What not to share

Never include the following in issues, discussions, or any public channel:

- Secrets or API keys
- Private logs or system data
- Customer data or corporate data
- Real personally identifiable information (PII)
- Proprietary code without permission

---

## Expectations

- There is no guaranteed response time.
- Maintainers and contributors help on a best-effort basis.
- Community is a support resource, not a dependency.
- If a repository is marked as a draft or experiment, expect incomplete documentation.
