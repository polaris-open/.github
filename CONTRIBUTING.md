# Contributing to Polaris

Thank you for your interest in contributing. Polaris is an early-stage open source community, and every thoughtful contribution helps.

---

## Values

- Clear writing
- Small, reviewable changes
- Practical examples
- Reproducible work
- Respectful disagreement
- Critical thinking
- Responsible use of AI
- Synthetic data in public learning projects

---

## Good first contributions

If you are unsure where to start, focus on:

- Typos and grammar
- Clearer explanations
- Practical examples
- Issue and PR templates
- Unclear or missing instructions
- Troubleshooting notes

---

## Before opening a PR

- Check the repository status and maturity label in the README.
- Check existing issues and PRs — avoid duplicates.
- Keep the change small and focused.
- Follow any guidelines specific to the repository.

---

## What a good PR includes

- **What changed** — a clear description of the modification.
- **Why it changed** — the motivation or problem it addresses.
- **How it was tested or reviewed** — how you validated the change.
- **Scope** — whether it affects docs, examples, or learning paths.

---

## AI-assisted contributions

AI tools are allowed. However:

- You are responsible for the content you submit.
- Do not submit content you do not understand.
- Do not submit generated output without reviewing it.
- Do not include private, proprietary, or sensitive data in prompts, commits, issues, or PRs.

---

## Synthetic data only

Public Polaris repositories are learning environments. This means:

- No real production logs
- No customer data
- No corporate data
- No credentials or secrets
- No API keys or tokens
- No real personally identifiable information (PII)

When examples require data — incidents, logs, support tickets, users, payments, orders — use clearly synthetic data.

---

## Community expectations

- Be respectful.
- Disagree with ideas, not people.
- Bring context when raising concerns.
- Avoid noise — prefer one clear message over multiple short ones.
- Prefer clarity over opinion when giving feedback.
